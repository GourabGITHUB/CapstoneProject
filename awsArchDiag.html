This is a 3 tier architecture solution for hosting a web application that receives client requests and uses MySQL to query data and send it back to client.

    The web application will be deployed to EC2, we will create 1 VPC with 2 public subnets and 4 private subnets. we will place two EC2instances inside a public subnet which will serve the front end interface or web tier of the architecture, we will be using multiple instances with two availability zones for more redundancy.

    we will create two more identical EC2instances in Auto Scaling Group to provide horizontal scalability during high traffic by adding instances as required, we can configure with a maximum 4 instances with desired of 3 and minimum 2 instances .

    Application Load Balancer will be used to distribute traffic to EC2instances, we will configure security groups with all the resources to control inbound and outbound traffic and manage permissions. we will create Auto Scaling Group with application and data tier independently

    For database we will be using RDS MySQL database, this will be running on a different instance than application tier and will replicate data in the second availability zone.

    We will use CloudWatch service for monitoring and optimizing resources, since we are using Auto Scaling Group, CloudWatch can monitor the instance usage and send alerts to Auto Scaling Group for adding instances and Application Load Balancer will then run health check and then new instances can receive traffic.
